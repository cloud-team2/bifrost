# Bifrost AI Agent 검증 결과 · 기준 · 출처 (2026-06-22)

운영/오프라인 환경에서 FastAPI Agent Server(`services/ai-service`)를 테스트한 **실측 수치**와, 그 평가 기준이 근거하는 **네임밸류 글로벌 표준·논문 출처**를 정리한다. 발표(PPT) 근거 문서.

> 수치 정직성 원칙: 아래는 **(A) 오늘 실제 측정 / (B) 코드·설계로 보장(측정 아님) / (C) 미측정·추가 필요**를 명확히 구분한다. 구버전 슬라이드의 `89.6% / 367케이스`는 코드에 출처가 없어 **사용하지 않는다**(재현 불가). 본 문서의 RCA 수치는 2026-06-22 재실행값이다.

## 2. 평가·설계 기준 — 글로벌 벤치마킹과 적용

우리 설계·평가 기준은 임의로 만든 게 아니라, **네임밸류 있는 글로벌 벤더·표준기구·논문이 쓰는 방식을 벤치마킹해 우리 프로젝트에 이식**한 것이다. 각 항목은 "**업계는 이렇게 한다(원리) → Bifrost는 이렇게 적용했다(구현)**" 형태로 정리한다. (URL은 리서치로 실재 확인. ISO/OpenAI 일부는 봇 차단으로 본문 미검증, 표기)

### 2.1 RCA 방법론 — IT 운영·품질관리 표준을 LLM RCA에 이식

**ITIL 4 Problem Management + KEDB** (Axelos 2020 · https://www.axelos.com/resource-hub/practice/problem-management-itil-4-practice-guide)
- 업계 원리: 인시던트의 근본원인을 분석한 뒤 **Known Error**로 등록(KEDB)해, 재발 시 즉시 진단·복구에 재사용한다.
- Bifrost 적용: 35종 root-cause 카탈로그를 KEDB처럼 운영 — 증상·검증된 조치·롤백 절차·과거 인시던트 링크를 보관(`app/catalogs/kedb.py`)하고 RCA가 이를 참조한다.

**5 Whys** (Toyota Production System, Ohno 1988 · https://en.wikipedia.org/wiki/Five_whys)
- 업계 원리: "왜?"를 반복해 표면 증상이 아닌 **진짜 근본원인**까지 내려간다.
- Bifrost 적용: 증상성 원인(예: Connector Task FAILED)이 떠도, 더 깊은 원인(예: Sink DB 연결 타임아웃)이 증거로 입증되면 증상을 그 아래로 **강등**해 근본원인을 top에 올린다(`rca.py` 증상 강등 로직).

**Ishikawa(특성요인도)** (ASQ · https://asq.org/quality-resources/fishbone)
- 업계 원리: 하나의 결과(문제)에 대한 잠재 원인을 **범주별로 분기**해 본다.
- Bifrost 적용: 1개 incident_type → 복수 root-cause 후보로 매핑(`incident_rootcause_map.py`)해 한 증상을 여러 원인 가설로 분기·점수화한다.

**ISO/IEC 20000-1:2018** (IT 서비스 관리 · https://www.iso.org/standard/70636.html)
- 업계 원리: incident → problem → known error로 이어지는 ITSM 프로세스를 체계화한다.
- Bifrost 적용: Classifier→RCA→Verifier→Report 파이프라인이 이 ITSM 흐름을 자동화한 형태다.

> 한 줄: "원인을 **생성**하지 않고 **카탈로그에서 증거로 선택**한다"는 RCA Assistant 원칙은, 수십 년 검증된 ITSM·품질관리 표준을 LLM에 그대로 이식한 결과다.

### 2.2 장애 알림·심각도 — Google SRE 운영 표준을 벤치마킹

**Google SRE — 4 Golden Signals** (https://sre.google/sre-book/monitoring-distributed-systems/)
- 업계 원리: 사용자 대면 시스템은 **Latency·Traffic·Errors·Saturation** 4개 신호를 본다.
- Bifrost 적용: 관측 도구 세트가 이 4신호를 그대로 커버 — 지표 조회(latency/saturation)·로그 검색(errors)·consumer lag 조회(traffic/saturation).

**Google SRE — Alerting on SLOs** (https://sre.google/workbook/alerting-on-slos/)
- 업계 원리: 알림은 단순 임계가 아니라 **error budget / burn rate**(추세) 기준으로, 잠깐의 스파이크엔 울리지 않는다.
- Bifrost 적용: 임계값을 숫자로 박지 않고 **value+version+basis+owner로 거버넌스**(`threshold_registry`)하고, WARN 단건은 로그만·동일 리소스 30분 2건↑일 때 인시던트로 승격해 알람 피로를 줄인다.

**PagerDuty Severity Levels** (https://response.pagerduty.com/before/severity_levels/) · **Confluent Kafka lag alerting** (https://docs.confluent.io/platform/current/monitor/monitor-consumer-lag.html)
- 업계 원리: 심각도를 SEV 등급으로 명확히 정의하고, Kafka lag은 절대값이 아니라 추세로 본다.
- Bifrost 적용: 심각도 2단계(WARNING/CRITICAL) + consumer lag 임계(WARNING 5,000 / CRITICAL 50,000, spec 부록 B 단일 출처).

### 2.3 AI 리스크·환각 통제 — 정부·업계·규제 프레임워크를 그대로 구현

**NIST AI RMF 1.0** (NIST AI 100-1, 2023 · https://nvlpubs.nist.gov/nistpubs/ai/nist.ai.100-1.pdf)
- 업계 원리: AI 위험을 GOVERN·MAP·**MEASURE**(정량 측정)·**MANAGE**(대응)로 관리한다.
- Bifrost 적용: RCA 정확도·ECE를 지표화해 MEASURE, 위험 조치를 승인 게이트로 MANAGE, 정책 매트릭스로 GOVERN.

**OWASP Top 10 for LLM 2025** (https://genai.owasp.org/llm-top-10/)
- 업계 원리: LLM01 Prompt Injection·LLM02 Sensitive Information Disclosure·LLM05 Improper Output Handling을 최우선 위협으로 규정한다.
- Bifrost 적용: 대화(ReAct) 루프엔 read-only 도구만 노출(인젝션으로 변경 불가), 자격증명·민감정보 자동 마스킹, 출력 요약 화이트리스트로 검증.

**EU AI Act 제14조 Human oversight** (Reg.(EU) 2024/1689 · https://artificialintelligenceact.eu/article/14/)
- 업계 원리(규제): 고위험 AI는 사용 중 사람이 **효과적으로 감독·재정의**할 수 있어야 한다.
- Bifrost 적용: 모든 변경 조치는 HITL 승인 게이트 통과 필수 — 미승인 시 백엔드 호출 자체가 차단.

**Google SAIF / Microsoft Responsible AI v2** (https://saif.google/ · Microsoft RAI Standard v2)
- 업계 원리: AI 고유 위협(인젝션·데이터 포이즈닝 등)에 대한 보안·책임성 프레임워크.
- Bifrost 적용: 인젝션 차단·출력 검증·HITL을 통해 책임성(accountability) 확보. (ISO/IEC 42001:2023 AI 경영시스템도 동일 맥락 — 운영·로깅·라이프사이클 거버넌스로 정렬)

### 2.4 정량 지표·기법 — 학술 표준 그대로

**ECE (Expected Calibration Error)** — Guo et al., ICML 2017 · https://arxiv.org/abs/1706.04599
- 업계 원리: "신뢰도 0.8이면 실제 정답률도 80%"가 되도록 보정 정도를 측정(10-bin).
- Bifrost 적용: ECE 0.083(<0.10)으로 과신 여부를 검증(`calibration.py`).

**RAG (검색증강생성)** — Lewis et al., NeurIPS 2020 · https://arxiv.org/abs/2005.11401
- 업계 원리: 외부 근거 문서를 검색해 조건으로 생성하면 환각이 준다.
- Bifrost 적용: pgvector 기반 KEDB·런북 검색으로 근거 기반 답변.

**Selective Prediction / Abstention** — Geifman & El-Yaniv, NeurIPS 2017 · https://arxiv.org/abs/1705.08500
- 업계 원리: 불확실한 입력은 **기권(reject)** 해 오류율을 보장한다.
- Bifrost 적용: 신뢰도 < 0.60이면 `UNKNOWN_WITH_EVIDENCE_GAP`로 기권(찍지 않음).

**Human-in-the-Loop** — Wu et al., FGCS 2022 · https://arxiv.org/abs/2108.00941
- 업계 원리: 인간 판단을 운영 루프에 통합해 정확성·안전성을 높인다.
- Bifrost 적용: 변경 조치 전 사람 승인(EU AI Act §14와 동일 취지).

**평가지표 표준** — Accuracy@k·Avg@5(RCAEval, arXiv:2412.17015) / MRR(Voorhees, TREC-8 1999) / Macro-F1(Sokolova & Lapalme 2009) / Wilson score interval(Wilson 1927; Brown et al. 2001) / LLM-as-judge(Zheng et al., NeurIPS 2023, 인간 일치 80%+)
- 업계 원리: RCA·분류 평가의 사실상 표준 지표.
- Bifrost 적용: AC@k·Avg@5·ECE를 산출(`metrics.py`·`calibration.py`). ※ Macro-F1·Wilson CI는 현재 미구현(추가 시 산출 가능).
