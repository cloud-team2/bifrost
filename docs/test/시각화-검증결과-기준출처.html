Bifrost AI Agent 검증 결과 · 기준 · 출처 (2026-06-22)

운영/오프라인 환경에서 AI 장애대응 에이전트(FastAPI Agent Server)를 테스트한 실측 수치와, 그 평가·설계 기준이 근거하는 글로벌 표준·논문을 벤치마킹해 적용한 내용을 정리한 문서.
RCA(근본원인분석) Assistant 원칙 — LLM이 원인을 생성하지 않고, 카탈로그·증거로 선택하고, 불확실하면 기권한다.

수치 정직성 원칙 — 본 문서는 A 오늘 실측 B 코드·설계로 보장 C 미측정·추가 필요를 명확히 구분한다. 구버전 슬라이드의 89.6% / 367케이스는 코드에 재현 출처가 없어 사용하지 않는다. 본 문서 RCA 수치는 2026-06-22 재실행값(보수적 floor).

2. 평가·설계 기준 — 글로벌 벤치마킹과 적용 B

우리 기준은 임의 설계가 아니라, 네임밸류 글로벌 벤더·표준기구·논문의 방식을 벤치마킹해 이식한 것이다. 각 항목 = "업계는 이렇게 한다(원리) → Bifrost는 이렇게 적용했다(구현)".

질문답
Q4. 로그만 취합해 LLM에 주면 RCA가 되나?아니오. 로그→증거 정규화→카탈로그 매칭→증거 게이트→점수화→검증의 결정론적 오케스트레이션. 로그만으론 단정 불가 → 기권
Q5. LLM 의존적 → 환각 대응은?5중 방어: 카탈로그-바운드 + 증거 게이트 + 신뢰도 임계 + 기권 + Verifier 재검증. ECE 0.083, 환각 0
Q6. rollback plan(실패 시나리오) 있나?있다. 조치 실패 시 역조치 자동(pause↔resume, restart→직전상태 복원). MEDIUM 자동복구, HIGH 승인 필요, 역조치 없으면 N/A
Q7. version 관리 되나?된다. run마다 재현성 매니페스트(model·prompt_hash·catalog·evidence_matrix·runbook·corpus·eval_dataset·code_commit_sha·temperature). DB alembic 18, 임계값 거버넌스
Q8. 장애 알림 기준?spec 부록 B 단일 출처: lag ≥5,000 WARN/≥50,000 CRIT, error rate >0.5%/>2%, Connector FAILED→CRIT, WARN 동일리소스 30분 2건↑. 4 Golden Signals 기반
